# Spec: 의도 분류기 (Intent Classifier)

## 목적
사용자의 건강 고민 혹은 질문 발화를 탐지하여 의도('research', 'welfare', 'general')를 분류합니다. 이를 통해 적합한 특화 에이전트로 작업을 라우팅하고, 대화의 품질 향상과 더불어 불필요한 LLM API 비용을 최소화(비용 0에 가까운 키워드 우선 해결)하는 것을 목적으로 합니다.

## 입력
| 파라미터 | 타입 | 필수 | 설명 |
| :--- | :--- | :--- | :--- |
| message | string | Yes | 사용자로부터 입력받은 채팅 텍스트 |

## 출력
| 필드 | 타입 | 설명 |
| :--- | :--- | :--- |
| intent | 'research' \| 'welfare' \| 'general' | 사용자 발화에서 감지된 최적의 에이전트 종류 |

## 처리 흐름 (단계별)
1. **1단계 (키워드 매칭 - 비용 0)**
   - 입력받은 발화를 소문자로 변환하여 키워드 매칭을 실행합니다.
   - `research` 키워드군: 논문, 연구, 임상, 치료, 효과, 최신, 약물, 병기, study, paper, clinical, evidence
   - `welfare` 키워드군: 복지, 지원, 혜택, 신청, 보험, 요양, 급여, 장애, 등록, 서비스, 자격
   - 두 키워드가 모두 발견되거나 `research` 키워드가 단 하나라도 발견되면 `research`로 분류합니다.
   - `welfare` 키워드만 있으면 `welfare`로 분류합니다.
2. **2단계 (GPT-4o-mini Few-shot Fallback)**
   - 키워드가 검출되지 않아 의도가 모호한 경우에만 GPT-4o-mini를 활용하여 Few-shot 프롬프트로 의도를 정밀 매칭합니다.
3. **3단계 (에이전트 미할당 Fallback)**
   - 에러가 나거나 분류가 불명확할 경우에는 비용 절약을 위해 `'general'`을 반환하여 일반 어드바이저가 즉각 응답합니다.

## 엣지케이스
- **동시 발생 키워드**: "만성 신부전 3기 논문 임상에 나온 치료 약물의 복지 혜택과 보험 요양이 궁금해"
  - 대처: `research` 키워드가 한 개라도 포함되면 의학 전문 정보가 중요하므로 `research`로 최종 결정합니다.
- **영어/한글 혼용 및 오탈자**: "클리니컬 스터디 있나요?"
  - 대처: 키워드 목록에 'study', 'clinical' 등을 포함하여 방어하고 2단계 Few-shot LLM에 처리 위임합니다.

## 성능 목표
- **정확도 (Accuracy)**: 90% 이상 (검증 셋 대상)
- **비용 최적화**: 80% 이상의 사용자 질의가 1단계 키워드 탐지 단계에서 0원에 해결되도록 설계.
- **처리 속도 (Latency)**: 키워드 탐지 시 5ms 미만, GPT fallback 시 300ms 내외.

## 테스트 케이스 (최소 5개)
1. **키워드 매칭 (Research)**:
   - 입력: "만성 콩팥병 최신 **임상 연구** 결과가 있나요?"
   - 기대결과: `research`
2. **키워드 매칭 (Welfare)**:
   - 입력: "당뇨 신장합병증 환자 **복지 지원금** 신청은 어디서 하나요?"
   - 기대결과: `welfare`
3. **키워드 혼합 우선순위**:
   - 입력: "투석 환자 **지원** 혜택 최신 **논문** 좀 찾아줘"
   - 기대결과: `research` (우선순위 규정에 따름)
4. **Few-Shot Fallback**:
   - 입력: "신장이 안 좋아졌는데 최신 치료 트렌드가 뭐죠?" (직접 키워드는 없으나 의도는 연구임)
   - 기대결과: `research` (LLM에 의해 잘 탐지됨)
5. **General 매칭**:
   - 입력: "오늘 점심 메뉴로 현미밥 먹어도 되나요?"
   - 기대결과: `general`
