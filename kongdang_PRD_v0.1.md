> **버전**: v1.1 | **작성일**: 2026-06-12 | **팀**: AIve (에이아이브)
> **대상**: 바이브코딩(Vibe Coding) 기반 개발자 온보딩 및 AI 에이전트 구현 명세

## 0. 문서 목적

이 PRD는 기존 `kongdang` 기본 챗봇 레포지토리에 **5개 전문 에이전트/워크플로우**를 추가하고, **Hybrid RAG** 파이프라인으로 빠르게 개발·배포하기 위한 바이브코딩용 개발 명세서입니다.

## 2. RAG 파이프라인 비교 평가 프레임워크 (배경 및 목표)

본 프로젝트는 복합 만성질환의 한국 표준 임상 지침과 복지 정보를 RAG(Retrieval-Augmented Generation) 시스템으로 서비스화하는 과정에서, 의미적(Ontology) 매핑과 미세조정(LoRA)이 RAG 성능에 미치는 효과를 정량적으로 비교 분석하는 것을 목표로 합니다.

### 2.1 3대 RAG 파이프라인 비교 대상

```
┌──────────────────────────────────────┐
│  Group A: RAG                         │
│  - 단순 텍스트 유사도 기반 정보 추출 및 답변 생성
└──────────────────┬───────────────────┘
                   │
                   ▼ (조건 추출 및 매칭 개선)
┌──────────────────────────────────────┐
│  Group B: RAG + Ontology              │
│  - 표준 의학 지식 온톨로지 매핑 및 다차원 조건부 필터링 연동
└──────────────────┬───────────────────┘
                   │
                   ▼ (의료 표현 및 구어체 매핑 이해 개선)
┌──────────────────────────────────────┐
│  Group C: RAG + Ontology + LoRA Fine-tuning │
│  - 구어체 의학용어 NER 파인튜닝 및 정밀 슬롯필링 관계 매핑 적용
└──────────────────────────────────────┘
```

### 2.2 5대 핵심 검증 포인트 (Verification Metrics)

사용자가 로그인한 상태에서 제공되는 **RAG 모델 비교표** 기능을 통해 다음 5가지 지표의 성능 향상 결과를 직관적으로 파악할 수 있도록 대화창에 비교 도구를 임베딩합니다:

1. **의도 분류 정확도 (Intent Accuracy)**: 구어체 질문에서 환자가 문의하고자 하는 임상/복지/식단/병원 영역을 완벽하게 추론하는 정확도.
2. **온톨로지 매핑 정확도 (Ontology Mapping Accuracy)**: 환자가 호소하는 다양한 의학 증상(예: "거품뇨", "소변 거품")을 표준 표준 용어(HPO, DOID 코드)로 정밀하게 연결하는 능력.
3. **관계 슬롯 추출 정확도 (Slot Filling Accuracy)**: 대화 과정에서 환자의 의도를 정확히 파악하여, 병원 추천에 핵심적이나 프로필에 누락된 필수 관계 슬롯(예: 현재 위치 정보, 야간/일반 투석 방식, 합병증 동반 상태 등)을 대화 흐름 중에서 정확히 채워 넣는 성능.
4. **전문 병원 추천 정확도 (Hospital Recommendation Accuracy)**: 위치, 투석 조건, 복합 합병증 동반 여부를 종합 반영하여 3차 상급종합병원과 일반 투석 의원을 적합하게 매칭 및 우선 정렬하는 추천 정확도.
5. **의학 정보 환각 비율 (Hallucination Rate)**: 임상 가이드라인과 사실 근거에서 벗어난 안전하지 않은 정보를 의학 상식 위배 없이 정확하게 제어하는 수준 (환각율 최소화).

```
사용자 입력
    ↓
의도 분류 라우터 (Intent Classifier)
    ↓
┌───────────────────────────────────────────┐

│  5. 병원 찾기 에이전트                        │
└───────────────────────────────────────────┘
    ↓
Hybrid RAG (Dense + Sparse 검색, Supabase + Pinecone)
    ↓
GPT-4o-mini 응답 생성
    ↓
사용자 출력 (웹 UI)
```

## 4. ⭐ RAG 데이터 넣는 방법 (비개발자용 안내)

> 수집한 PDF, 논문, 가이드라인 파일을 어디에 넣으면 되는지 안내합니다.

### 폴더 경로 및 분류 기준

| 폴더 경로 | 넣을 파일 종류 | 예시 |
|-----------|--------------|------|
| `data/raw/ckd/` | 콩팥병 관련 PDF, 가이드라인 | 대한신장학회 진료지침 PDF, 투석 안내서 |
| `data/raw/dm/` | 당뇨병 관련 PDF, 가이드라인 | 대한당뇨병학회 가이드라인, 인슐린 설명서 |
| `data/raw/welfare/` | 복지·보험 관련 문서 | 장애인 복지급여 안내, 건강보험 산정특례 PDF |
| `data/raw/diet/` | 식단·영양 관련 문서 | 식약처 식품성분표, 투석식 레시피북, 한식 영양분석 |
| `data/raw/hospital/` | 병원·의료기관 정보 | 투석 가능 병원 목록, 전문 클리닉 안내 |
| `data/raw/research/` | 연구·신약 관련 문서 | PubMed 논문 PDF, 임상시험 안내문 |

### 지원 파일 형식
- ✅ PDF (`.pdf`) — 가장 권장
- ✅ Word 문서 (`.docx`)
- ✅ 텍스트 (`.txt`)
- ✅ 엑셀/CSV (`.xlsx`, `.csv`) — 표 형태 데이터
- ❌ 스캔 이미지 PDF는 별도 OCR 처리 필요 (담당자에게 문의)

### 파일 넣은 후 할 일
```bash
# 개발자가 아래 명령 한 번 실행하면 자동으로 Pinecone에 업로드됨
python scripts/ingest.py --folder data/raw/
```

## 5. 5개 에이전트 상세 명세

### 5.1 증상문진 에이전트 (Symptom Agent)

| 항목 | 내용 |
|------|------|
| **목적** | 한국어 구어체 증상 표현을 정확히 이해하고 임상 개념으로 매핑 |
| **트리거 키워드** | "아파요", "불편해요", "거품뇨", "발저림", "피곤해요", "붓는 것 같아요" |
| **핵심 기능** | 구어체→HPO/DOID 코드 매핑, 응급 증상 즉각 감지, 위험도 판정 |
| **응급 우선 처리** | 저혈당, 고칼륨혈증, 급성 신손상 키워드 감지 시 즉시 응급 프로토콜 출력 |
| **RAG 소스** | `data/raw/ckd/`, `data/raw/dm/` |
| **출력 형식** | 위험도 (정상/주의/위험/응급) + 대처 가이드 + 병원 방문 권고 여부 |

#### 워크플로우 상세 명세

```
┌─────────────────────────────────────────────────────────────┐
│  STEP 1. 사용자 증상 입력                                     │
│  예) "다리가 퉁퉁 붓고 소변에 거품이 많이 나요"                  │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 2. 🚨 응급 키워드 즉시 필터 (Rule-based, <0.1초)        │
│  감지 키워드: 의식 잃다 / 숨 못 쉬다 / 혈당 30 이하 /          │
│             심한 가슴통증 / 갑자기 소변 안 나와                  │
│  ─────────────────────────────────────────────              │
│  [응급 감지 시] → 즉시 응급 프로토콜 출력 후 흐름 종료           │
│   "119 즉시 신고 / 응급실 방문" + 대기 중 대처법 안내            │
│  [응급 아닐 시] → STEP 3으로 진행                              │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 3. 한국어 구어체 → 표준 의학 용어 매핑 (NER + LoRA)      │
│                                                             │
│  구어체 입력           →  표준 매핑                            │
│  "소변에 거품"         →  단백뇨 HPO:0000093                  │
│  "다리 퉁퉁 부음"      →  부종(edema) + CKD 수분저류           │
│  "손발 저림"           →  당뇨신경병증 DOID:3070               │
│  "머리가 띵함"         →  요독증 전조 / 저혈당 감별 필요         │
│  "소변이 뿌옇다"       →  혼탁뇨 → 감염 또는 인산염뇨 감별       │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 4. 사용자 질환 프로필 컨텍스트 주입                       │
│  마이페이지 프로필을 자동으로 불러와 함께 전달                    │
│  예) CKD 4기 + 2형 당뇨 + 혈액투석 중 + ARB 복용 중            │
│  → 증상 해석 시 해당 단계/상태에 맞는 기준값 적용               │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 5. Hybrid RAG 검색 (임상 가이드라인)                    │
│  검색 네임스페이스: symptom (ckd + dm 가이드라인)              │
│  쿼리: 매핑된 표준 의학 용어 + 환자 프로필                      │
│  반환: 관련 임상 지침 청크 Top-5 (출처 메타데이터 포함)          │
│  예) 대한신장학회 CKD 진료지침 2023, p.47 부종 관리 기준        │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 6. GPT-4o-mini 위험도 판정 + 응답 생성                  │
│                                                             │
│  판정 기준:                                                  │
│  🟢 정상   — 기준치 내, 경과 관찰                              │
│  🟡 주의   — 기준치 근접, 1주 내 외래 권고                      │
│  🔴 위험   — 기준치 초과, 빠른 외래/입원 권고                   │
│  🚨 응급   — 즉시 119 / 응급실 (STEP 2와 별개로 2차 감지)       │
│                                                             │
│  응답 구성 요소:                                              │
│  1) 위험도 뱃지 (색상 + 등급)                                  │
│  2) 증상 해석 (표준 용어 + 쉬운 설명 병기)                      │
│  3) 즉시 대처법 (단계별 행동 지침)                              │
│  4) 병원 방문 권고 여부 + 방문 시 말할 말                       │
│  5) 근거 출처 (가이드라인명 + 발행기관)                          │
│  6) 추가 확인 질문 제안 2~3개                                  │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 7. 출력 + 후속 액션 버튼                                │
│  [병원 찾기 에이전트로 연결] [식단 에이전트로 연결]              │
│  [대화 저장] [⚠️ 면책 고지 자동 첨부]                          │
└─────────────────────────────────────────────────────────────┘
```

**프롬프트 핵심 지침**:
```
- 환자가 "다리가 퉁퉁 부어요"라고 하면 → 부종(edema) + CKD 수분저류 가능성 추론
- "오줌에 거품이 너무 많이 나요" → 단백뇨(proteinuria) HPO:0000093 매핑
- "손발이 저려요" → 당뇨신경병증(DOID:3070) 또는 요독성 말초신경병증 감별
- 항상 "지금 당장 응급실에 가야 하나요?" 질문에 명확히 Yes/No로 먼저 답할 것
- 복합 증상일 경우 CKD와 DM 두 질환 관점에서 모두 해석할 것
```

**한국어 구어체 매핑 사전 (핵심 30개)**:

```
│  쿼리: 식품명 or 운동 유형 + 환자 CKD 단계 + 투석 여부          │
│  반환: 관련 식이요법 지침 청크 Top-5                           │
│  예) 대한신장학회 투석 환자 영양 관리 지침 2024                 │
│      대한당뇨병학회 식사 요법 권고안 2023                       │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 5. GPT-4o-mini 응답 생성                               │
│                                                             │
│  식품 판정 응답 구성:                                         │
│  1) 판정 결과 뱃지 (🟢🟡🔴⛔)                                │
│  2) 성분별 수치 + 이유 설명 (칼륨 OOmg — 기준치 OOmg 초과)    │
│  3) 안전한 섭취량 or 금지 이유                                 │
│  4) 대안 식품 2~3가지 (한식 기반)                              │
│  5) 조리 팁 (저칼륨 처리법: 껍질 제거, 물에 2시간 담그기 등)     │
│  6) 근거 출처 (가이드라인명)                                   │
│                                                             │
│  운동 응답 구성:                                              │
│  1) 추천 운동 종류 + 강도 + 시간                               │
│  2) 피해야 할 운동 (이유 포함)                                 │
│  3) 저혈당 대비 간식 타이밍                                    │
│  4) 운동 전후 체크리스트                                      │
└────────────────────────┬────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────────────────┐
│  STEP 6. 출력 + 후속 액션 버튼                                │
│  [1주일 식단표 생성하기] [증상 에이전트로 연결]                 │
│  [식품 성분 더 검색하기]                                      │
└─────────────────────────────────────────────────────────────┘
```

### 5.5 병원 정보 에이전트 (Hospital Agent - 구 병원 찾기)

| 항목 | 내용 |
|------|------|
| **목적** | 한국 특화 CKD·DM 전문 의료기관 정보 제공 및 대화 내 대화형 탐색 |
| **트리거 키워드** | "병원", "어디 가야", "투석 가능한 곳", "신장내과", "내분비내과", "가까운", "병원 찾기" |
| **핵심 기획** | 외부 탭/화면 분리가 아닌, **동일한 통합 챗봇(ChatWindow) 대화 내**에서 의도(Intent) 탐지를 통해 트리거되어 실행됨. |
| **위치 정보 획득 방식** | 대화 내 브라우저 GPS Geolocation 요청 및 거부 시 수동 지역 입력(Slot-Filling) 연동 |
| **출력 형식** | 대화 버블 내부의 **인라인 인터랙티브 위젯(HospitalSearchTab)** |

#### 5.5.1 상세 대화형 위치 탐색 (Slot-Filling) 워크플로우

1. **위치 허용 시 (GPS 동의)**
   - 사용자가 위치 정보 제공 동의 시, 브라우저 API를 활용해 현재 위/경도 좌표를 획득합니다.
   - 현재 위치 기준으로 반경 5km 이내의 인근 혈액/복막/야간 투석실 보유 병원 3개를 추천 리스트로 즉시 매칭하고, 예상 거리(예: "1.2 km")를 함께 표출합니다.
2. **위치 거부 시 (GPS 거절/차단)**
   - 사용자가 권한을 거부하면, 챗봇 대화방 내에 수동으로 지역명을 검색할 수 있는 입력 필드(Slot-filling UI)가 활성화됩니다.
   - 사용자가 "강남구", "동작구" 등 거주 지역이나 역명을 텍스트로 입력하면, 해당 검색 조건에 해당하는 지역 내 투석 병원 목록을 조회 및 필터링하여 출력합니다.

#### 5.5.2 인라인 병원 카드 컴포넌트 명세

* **핵심 투석 정보 표출**:
  - 각 추천 병원 카드에는 `혈액투석`, `복막투석`, `야간투석` 여부가 직관적인 체크마크(`✓ 가능` / `✗ 미운영`) 형태로 표시됩니다.
* **이동성 및 행동 편의성 제안**:
  - `[전화]` 버튼: 클릭 시 모바일 기기에서는 바로 전화를 연결하거나, 웹에서는 가상 연락 팝업/확인창을 제공합니다.
  - `[지도]` 버튼: 클릭 시 해당 병원 명칭을 쿼리로 네이버 지도(Naver Maps) 검색 페이지를 새 창(`_blank`)으로 열어 대중교통 경로 등을 신속히 확인할 수 있도록 딥링크를 지원합니다.
* **병원 기본 내원 주의사항**:
  - 인공신장실 선정 시 신장내과 분과 전문의 상주 여부, 전담 간호사 상주 여부, 응급 처치 연계 체계(심장/뇌혈관)를 확인해야 한다는 정보를 항시 노출합니다.

#### 5.5.3 합병증 여부 확인 및 재조정 워크플로우

* **합병증 동반 여부 확인 버튼**:
  - "신장·당뇨 합병증 동반 여부 확인하기" 버튼을 인터랙티브하게 노출합니다.
* **병원 목록 자동 재조정**:
  - 사용자가 합병증 버튼을 클릭하면, 병원 목록을 즉시 갱신(정렬/필터링)하여 다학제 진료가 가능한 3차 **상급종합병원**을 우선순위 최상단으로 올리고, 카드에 보라색 **[합병증 특화]** 뱃지를 표시합니다.
* **추가 주의사항 패널 노출**:
  - 합병증 동반 활성화 시, 붉은색 경고 박스로 **[🚨 합병증 발생 환자 필수 추가 주의사항]** 패널이 동적으로 노출됩니다.
  - 내용: 뇌졸중, 심근경색, 당뇨망막증, 당뇨발 등 합병증 동반 시에는 이식 센터 및 응급 야간 투석실을 갖춘 3차 대학병원으로 전원해야 하며, 의료보험 급여 혜택을 온전히 받으려면 **1차/2차 병원에서 발급한 진료의뢰서(요양급여의뢰서)**와 최근 검사기록지를 반드시 지참해야 함을 명시합니다.

**병원 유형 안내 기준 (한국 특화)**:

| 상황 | 추천 병원 유형 | 이유 |
|------|-------------|------|
| 투석 시작 전 상담 | 상급종합병원 신장내과 | 투석 방법 결정, 혈관 접근로 수술 |

| 야간 투석 필요 | 야간 투석 운영 의원 | 직장인, 낮 투석 불가 환자 |

## 6. Hybrid RAG 파이프라인 명세

### 6.1 아키텍처

```
문서 수집 (PDF/DOCX)
    ↓
[전처리] 청크 분할 (500토큰, 50 overlap)
    ↓
[임베딩] BGE-M3 → Dense Vector
[키워드] BM25 → Sparse Vector  
    ↓
[저장] Pinecone (벡터) + Supabase (메타데이터)
    ↓
[검색] 사용자 쿼리 → Dense + Sparse 검색 병렬 실행
    ↓
[융합] RRF(Reciprocal Rank Fusion)로 결과 합산
    ↓
[리랭킹] CrossEncoder로 Top-K 재정렬
    ↓
[생성] GPT-4o-mini + Retrieved Chunks → 최종 응답
```

### 6.2 청크 전략

| 문서 유형 | 청크 크기 | Overlap | 메타데이터 |
|-----------|----------|---------|-----------|
| 임상 가이드라인 PDF | 500 토큰 | 50 토큰 | 출처, 발행연도, 질환유형, 페이지 |
| 식품 성분표 CSV | 행 단위 | 없음 | 식품명, 성분, 단위 |
| 복지 안내문 | 300 토큰 | 30 토큰 | 기관명, 혜택유형, 자격조건 |
| 연구 논문 | Abstract 단위 | 없음 | PMID, 저자, 연도, 키워드 |

## 7. 화면 구성 명세 (UI)

### 7.1 메인 채팅 화면 `/chat` (단일 통합 챗봇 인터페이스)

기존의 개별 카테고리 탭이나 챗봇 외부 카드를 전면 배제하고, **대화창 단일 인터페이스**로 통합하여 설계합니다.

```
┌──────────────────────────────────────────────────┐
│  ❤️ 콩당콩당     AI 챗봇    건강 대시보드     [로그인] │  ← GNB (로고에 콩 이미지 배제, 심플&모던)
├──────────────────────────────────────────────────┤
│ ┌──────────┐┌──────────────────────────────────┐ │
│ │          ││                                  │ │
│ │  상담    ││   대화 시작하기 (Empty State)    │ │
│ │  이력    ││  ┌───────────────────────────┐   │ │
│ │  목록    ││  │ 🩺 증상 문진 시작하기     │   │ │
│ │  (LNB)   ││  └───────────────────────────┘   │ │
│ │          ││  ┌───────────────────────────┐   │ │
│ │  *로그인 ││  │ 🏥 주변 병원 정보 찾기    │   │ │
│ │  시에만  ││  └───────────────────────────┘   │ │
│ │  표출    ││                                  │ │
│ │          ││  ────────────────────────────────│ │
│ │          ││   [메시지 입력창...]   [전송 ▶]  │ │
│ └──────────┘└──────────────────────────────────┘ │
├──────────────────────────────────────────────────┤
│ [관리콘솔] | 서비스 소개 | 이용약관    (FOOTER)  │  ← footer로 관리콘솔 이동
└──────────────────────────────────────────────────┘
```

#### 7.1.1 레이아웃 주요 설계 규칙
1. **GNB (Global Navigation Bar)**:
   - 로고: '콩당콩당' 텍스트와 심플한 미소 짓는 하트 모양 심볼로 모던하고 심플하게 구성 (콩🫘 이미지 제외).
   - 메뉴 구성: `콩당콩당 (홈)`, `AI 챗봇`, `건강 대시보드`, `로그인 / 로그아웃`.
   - 불필요한 '트렌드' 페이지는 삭제/숨김 처리.
2. **LNB (Left Navigation Bar - 대화 이력)**:
   - 로그인 상태: 해당 사용자 본인의 지난 대화 목록 히스토리가 왼쪽 LNB 영역에 노출됨.
   - 비로그인(게스트) 상태: 대화 목록을 표시하지 않으며, 챗봇 대화창이 전체 너비로 시원하게 배치됨.
3. **챗봇창 내부 UI 설계**:
   - **대화창 외부 카테고리 카드 삭제**: `Health Workflows` 5개 카드는 UI 복잡도를 낮추기 위해 삭제함.
   - **대화 시작 화면 (Empty State)**: 깔끔한 인트로 문구와 함께 2개의 퀵 버튼만 배치하여 사용을 유도함.
     - `🩺 증상 문진 시작하기`
     - `🏥 주변 병원 정보 찾기` (병원 에이전트와 위치 슬롯필링 즉시 트리거)
   - **스트리밍 답변 및 길이 제한**:
     - 대화 답변은 가시성과 신속성을 위해 **700자 이내**로 제한하되 필수 임상 주의사항은 무조건 포함함.
     - 답변 하단에 다음 턴의 맥락 유지를 유도하는 추가 정보 탐색 버튼이나 출처 가이드를 깔끔히 배치함.
4. **인라인 에이전트 위젯**:
   - 병원 정보 검색 등 사용자 조작이 가미된 태스크는 대화 도중에 **메시지 버블 아래에 동적 React 위젯(HospitalSearchTab)**으로 삽입되어 대화 맥락을 끊지 않고 자연스러운 위치 슬롯필링 및 필터링 제어가 가능함.
5. **Footer**:
   - GNB에서 접근하던 "관리콘솔" 메뉴를 Footer로 이동하여 화면 집중도를 높임.

| `/admin` | 관리콘솔 / 관리자 | 에이전트 관리 및 지표 모니터링 (Footer 링크로 진입). |

### 7.3 질환 프로필 & 권장 섭취 제한량 (대시보드 필수 입력 항목)

건강 대시보드(`/dashboard`)에서 사용자의 개인별 데이터를 입력하면 eGFR이 자동 계산되며, 신장 기능 상태에 따른 당/나트륨/칼륨/인 제한량이 설정되고 이 정보가 챗봇 컨텍스트로 전달됩니다.

1. **기본 신체 및 검사 프로필**:
   - **성별**: 남성 / 여성 (eGFR 계산 보정 계수로 활용)
   - **나이**: 연령 입력 (eGFR 계산 공식 적용)
   - **키**: 신장(cm) 입력
   - **건체중 (목표 체중)**: 환자의 일일 목표 체중(kg) 입력
   - **최근 혈액검사 크레아티닌 (Cr)**: mg/dL 단위의 수치 입력
   - **eGFR 산출 공식 (MDRD)**: $175 \times \text{Cr}^{-1.154} \times \text{Age}^{-0.203} \times (0.742 \text{ if female})$ 식을 통해 사구체여과율 자동 계산 및 병기 단계 자동 매핑.
2. **영양 제한 섭취량 가이드라인**:
   - **당 (Sugar)**: 당뇨 동반 여부에 따라 30g~50g 제한.
   - **나트륨 (Sodium)**: 고혈압 및 수분저류 관리를 위해 2000mg 제한.
   - **칼륨 (Potassium)**: 신장 병기 3기 이상 2000mg, 4기/5기/투석 1500mg 수준으로 차등 자동 제한.
   - **인 (Phosphorus)**: 신장 병기 3기 이상 800mg~1000mg 차등 조절.
